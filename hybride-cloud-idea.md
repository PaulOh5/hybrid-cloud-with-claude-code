# 하이브리드 GPU 클라우드
규모에 상관없이 GPU가 장착된 linux 시스템을 쉽게 클라우드 노드로 만들어주는 하이브리드 GPU 클라우드 플랫폼

## 1. 배경
1. 개인에게 AI 연구용 multi gpu 워크스테이션을 판매하는 사업을 하고 있음
2. 이젠 개인에게 판매하던 multi gpu 워크스테이션을 사용해서 중,소형급의 데이터센터를 지을 계획임
3. 이 때, 개개인에게 판매한 워크스테이션과 데이터센터를 한 번에 묶을 클라우드 플랫폼이 필요 (하이브리드 클라우드)

## 2. 기능
### 2-1. GPU 소유자
1. 자신이 가진 GPU 워크스테이션에 프로그램을 설치하여 다른 사람에게 gpu 인스턴스를 제공할 수 있음
2. 데이터센터 관리자도 모든 GPU Node에 똑같은 프로그램을 설치하여 gpu 인스턴스 제공 가능
3. 인스턴스 제공자는 자신만의 zone을 만들고 zone에 GPU Node를 추가할 수 있음
4. gpu 인스턴스는 linux의 kvm을 기반으로 VM 형태로 제공

### 2-2. GPU 사용자
1. 하이브리드 클라우드 플랫폼 웹에 접속하여 다른 사람이 만들어놓은 zone에서 gpu 인스턴스를 생성할 수 있음
2. 생성된 gpu 인스턴스에 도메인으로 ssh 접속 가능 (ex. {random}.hybrid-cloud.com)

## 3. 대략적인 아키텍처
1. GPU 워크스테이션에서 gpu 인스턴스를 관리(생성 및 삭제)하는 프로그램 - compute-agent
2. compute-agent가 연결되는 backend 서버 - main-api
3. 사용자의 ssh 접속 도메인을 확인하여 올바른 compute-agent로 전달하는 proxy 서버 - ssh-proxy
4. zone 상태, 인스턴스 접속정보, 로그인같은 기본적인 서비스를 제공하는 웹 - frontend
